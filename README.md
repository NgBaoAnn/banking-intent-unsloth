# Banking Intent Classification — BANKING77

Fine-tune **Qwen2.5-7B** on the [BANKING77](https://huggingface.co/datasets/mteb/banking77) dataset (77 intent classes) using **Unsloth + QLoRA (4-bit)**.

## 4. Data Preprocessing

Downloads `mteb/banking77` from HuggingFace, normalizes text (lowercase, remove special characters), and splits into train / valid / test CSV files.

```bash
python scripts/preprocess_data.py
```

**Optional arguments:**

| Argument | Default | Description |
|---|---|---|
| `--samples_per_class_train` | `80` | Training samples per class |
| `--samples_per_class_valid` | `10` | Validation samples per class |
| `--samples_per_class_test` | `30` | Test samples per class |
| `--output_dir` | `sample_data` | Output directory |
| `--seed` | `42` | Random seed |

**Example (custom sampling):**

```bash
python scripts/preprocess_data.py \
    --samples_per_class_train 100 \
    --samples_per_class_valid 15 \
    --samples_per_class_test 30 \
    --output_dir sample_data
```

**Output files:**

| File | Description |
|---|---|
| `train.csv` | Training set (~6160 samples at default) |
| `valid.csv` | Validation set (~770 samples) |
| `test.csv` | Test set (~2310 samples) |
| `label_map.csv` | Label name ↔ ID mapping (77 intents) |

## 5. Training

### One-command (recommended)

Runs preprocess → fine-tune → evaluate in sequence:

```bash
bash train.sh
```

### Step-by-step

```bash
# Step 1: Preprocess data
python scripts/preprocess_data.py

# Step 2: Fine-tune + evaluate
python scripts/train.py \
    --config configs/train.yaml \
    --eval_config configs/evaluation.yaml
```

### Training hyperparameters (`configs/train.yaml`)

| Parameter | Value |
|---|---|
| Model | `unsloth/Qwen2.5-7B` |
| Quantization | QLoRA (4-bit) |
| Max sequence length | 512 |
| LoRA rank / alpha | 64 / 64 |
| Batch size | 2 (× 4 grad accum = 8 effective) |
| Learning rate | 5e-5 |
| Optimizer | AdamW 8-bit |
| Epochs | 6 |
| Warmup steps | 20 |
| Weight decay | 0.01 |
| LR scheduler | Cosine |

### Output

- **Checkpoint:** `./outputs/intent-model/`
- **Evaluation results:** `./outputs/intent-model/evaluation_results.txt`
  - Overall accuracy
  - Per-class Precision / Recall / F1 (Classification Report)

## 6. Inference

### Via shell script

```bash
# Single message
bash inference.sh "I am still waiting on my card?"

# Demo mode (runs multiple example messages)
bash inference.sh
```

### Via Python

```python
from scripts.inference import IntentClassification

classifier = IntentClassification("configs/inference.yaml")

print(classifier("I am still waiting on my card?"))   # → "card_arrival"
print(classifier("How do I change my pin number?"))   # → "change_pin"
print(classifier("Why was I charged a fee?"))         # → "extra_charge_on_statement"
```

### Via CLI

```bash
python scripts/inference.py \
    --config configs/inference.yaml \
    --text "I am still waiting on my card?"
```

> **Note:** Make sure the checkpoint exists at the path specified in `configs/inference.yaml` (`model_path`) before running inference.

## 7. Configuration Reference

### `configs/train.yaml`

```yaml
model_name: "unsloth/Qwen2.5-7B"
max_seq_length: 512
dataset_path: "./sample_data"
lora_r: 64
lora_alpha: 64
batch_size: 2
gradient_accumulation_steps: 4
learning_rate: 5.0e-5
num_train_epochs: 6
optimizer: "adamw_8bit"
warmup_steps: 20
weight_decay: 0.01
lr_scheduler_type: "cosine"
output_dir: "./outputs/intent-model"
```

### `configs/inference.yaml`

```yaml
model_path: "./outputs/intent-model"   # Path to saved checkpoint
max_seq_length: 512                    # Must match the value used during training
```

### `configs/evaluation.yaml`

```yaml
test_dataset: "./sample_data/test.csv"
output:
  results_file: "evaluation_results.txt"
confidence_threshold: 0.5
```
